# The Bench - Portal User Guide

This document provides concise operational instructions for Admin, Judge, and User accounts in The Bench Legal Portal.

---

## 1. Landing Page and Login

- Access the portal via the live URL.
- Unregistered visitors can view the basic landing page but cannot lodge a case or lodge arguments.
- Users must use the dedicated Login page to authenticate session access.
- Registered accounts remain pending until authorised by the Registrar.

---

## 2. Admin Portal (Registrar's Office)

The Admin functions strictly as the Court Registrar.

- **Role Requirement**: Authenticate registration approvals only.
- **Pending Registrations**: Approve or Reject new account requests.
- **Cases View**: Read-only tracking of active and pending docket listings with presiding judge and summon metadata.
- **Restrictions**: Admin cannot create, allocate, or resolve legal cases.

---

## 3. Judge Portal (The Bench)

Presiding judges maintain supreme authority over active trial proceedings.

### Review Board
- **Admit Applications**: Read filed arguments and formally admit cases into active court trial (Case becomes Active).
- **Reject applications**: Dismiss frivolous filings.

### Case Actions
- **Intervene**: Post bench directives and clarification questions direct to the Court log timeline.
- **Assign Stakeholder**: Select registered users and define custom case-specific roles (e.g., Plaintiff, Accused, Witness).
- **Issue Summon**: Insert court directives for appearance targeting specific names and purposes.

### Studio & Profile
- **Creator Studio**: Publish notices and legal literature to the landing feed with attached appendix files.
- **Profile**: Set qualifications and landmark history for disclosure to the public landing page profile directory.

---

## 4. User Portal (My Case Repository)

Every registered user maintains a central repository outlining their legal docket.

- **My Cases**: Grid list rendering your active or pending case refs. Select a case to activate timeline feeds and load argument actions.
- **Summons**: Direct view panel consolidating all court summons issues against you across all items with pending/fulfilled flags.
- **Documents & Proofs**: Automated aggregate bucket containing all filed arguments and submitted proof records time-stamped date-wise per Selected case.
- **File Argument**: Dedicated textarea to submit motion entries targeting the currently selected sidebar case for Judge verification.

---

## Technical Support / Logic Audit
To report bugs or discrepancies to developers, select the Floating Action button located bottom right on Dashboard frameworks to broadcast Instant report alerts direct to the Engineering response team.
